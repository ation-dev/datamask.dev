FAQs#

Where does Datamask run?#

Datamask runs inside environments you control.

It can be deployed as:

  • a CLI
  • a container (Kubernetes Job or CronJob)
  • or inside a CI runner

Does Datamask need public internet access?#

No.

Datamask can run in:

  • private networks
  • restricted VPCs
  • or air-gapped environments

How does Datamask access our databases?#

Datamask connects using credentials provided by your existing secret management system.

Secrets are never stored in configuration files or written to disk.
The configuration references secrets by logical name, and the runtime resolves them from your environment.

Who writes the configuration?#

In early versions, configurations are written by engineers.

Datamask is designed to assist with:

  • detecting likely sensitive fields
  • suggesting strategies
  • and generating draft configurations

But execution always requires explicit, human-approved configuration.

What happens if our schema changes?#

Datamask detects schema drift by comparing the source schema to what the configuration expects.

You can choose to:

  • fail the run
  • or require explicit review

This prevents silent data corruption when production schemas evolve.

Can Datamask accidentally expose real data?#

Datamask is designed to fail safe.

If:

  • a column is not covered by rules
  • a constraint cannot be met
  • or a safety condition is violated

the run stops instead of producing unsafe output.

How is this different from copying production data?#

Datamask produces:

  • safe, transformed values
  • preserved relationships
  • stable identities
  • and a full audit trail

You get production-like behaviour without exposing production data.

Can we reproduce a dataset later?#

Yes.

Every Datamask run is tied to:

  • a configuration
  • an engine version
  • and an input schema

The same inputs and configuration will always produce the same outputs.

Can this be automated?#

Yes.

The same configuration used for manual runs can be executed in:

  • CI pipelines
  • scheduled jobs
  • or environment refresh workflows

Automation does not change behaviour â€” it only changes how the run is triggered.

Is Datamask replacing our data platform or pipelines?#

No.

Datamask integrates with:

  • your databases
  • your secret managers
  • your CI/CD
  • and your storage

It provides a deterministic execution layer for non-production data, not a new data platform.

What is expected from a design partner?#

Design partners typically:

  • provide a real schema or environment
  • define one or two target use cases (e.g., UAT refresh, QA dataset)
  • and give feedback on usability and data quality

The goal is to shape Datamask around real engineering needs.
